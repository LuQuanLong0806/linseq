## 任务目标

仅对**本次变更（新增/修改）的文件**完成全栈开发：

- 前端：使用现有 TailwindCSS + Three.js 实现科技感炫酷页面，代码精简、省 Token
- 后端：自动补充/修改对应接口、服务、数据结构、路由
- 全量：增量单元测试 + 语法检测 + 格式校验 + 自动运行与修复
  适用：刚改完代码、增量提交、小功能迭代、全栈页面开发。

## 触发句式

- 做科技感3D页面 + 配套后端接口
- 前端页面开发完成，补充对应后端逻辑
- 全栈修改，增量单测，前后端一起写完
- 只处理本次变更，前后端+测试全覆盖
- 开发页面并生成对应接口与单元测试

## 执行流程（必须严格按顺序）

1. **获取变更范围**
   - 列出本次新增/修改文件（不含删除）
   - 只处理源码文件（.js/.ts/.vue/.react/.py/.java/.go 等），跳过配置/文档
   - 自动识别：前端文件 + 对应后端文件

2. **语法与格式检测（强制）**
   - 前端 JS/TS：eslint + tsc --noEmit
   - 后端：对应语言编译/语法检查
   - 有错误：先修复语法/格式，再继续；禁止生成带语法错的代码/测试

3. **前端开发（强制约束）**
   - 技术栈：仅使用项目现有 TailwindCSS + Three.js
   - 不新增依赖、不改动构建配置、不破坏现有工程结构
   - 视觉：科技感、3D动效、霓虹、磨砂玻璃、流光、粒子背景
   - 代码：极简、无冗余、省 Token、结构清晰
   - 交互：流畅、响应式、高性能

4. **后端同步开发（强制）**
   - 根据前端页面需求，自动补充/修改：
     - 接口路由 / API 地址
     - 请求/返回数据结构
     - 业务服务逻辑
     - 参数校验、异常处理
   - 保持与现有后端架构、命名、风格完全一致
   - 不随意新增库、不破坏原有服务逻辑

5. **分析代码与全链路**
   - 提取前后端所有公开函数、方法、接口、类、组件
   - 梳理全链路：前端请求 → 后端接口 → 数据返回 → 前端渲染
   - 覆盖：正常场景、边界值、空值、非法输入、异常抛出
   - 如有Agent任务用到的相关的变更修改及时更新 docs/QCLAW_AGENT_GUIDE.md 为Agent开发提供正确、详细的文档说明

6. **生成单元测试（覆盖要求）**
   - 框架：自动沿用项目现有（Jest/Vitest/pytest/JUnit）
   - 范围：仅本次修改/新增文件
   - 必须覆盖：
     - ✅ 前端：DOM渲染、样式加载、3D场景、动画、请求调用
     - ✅ 后端：接口入参、出参、业务逻辑、异常、边界
     - ✅ 全链路：前端调用 + 后端返回
   - Mock：外部服务、数据库、第三方接口必须 Mock

7. **写入并运行测试**
   - 生成可直接运行的测试文件，不改动源码
   - 自动执行测试命令
   - 失败：自动修复断言、Mock、逻辑，直到通过
   - 输出：前后端测试结果 + 覆盖率摘要

## 输出格式

1. 变更文件清单（前端 + 后端）
2. 语法/格式检查结果
3. 前端完整代码（科技感 Three.js + TailwindCSS）
4. 后端对应代码（接口/服务/逻辑）
5. 单元测试代码（前后端全覆盖）
6. 运行命令 + 执行结果 + 覆盖率
7. 如失败：问题说明 + 自动修复方案
